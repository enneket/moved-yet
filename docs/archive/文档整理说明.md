# 文档整理说明

## 整理目标

将 docs 目录从 50+ 个混乱的文档整理为清晰的 3 级分类结构。

## 最终结构

```
docs/
├── README.md                    # 📚 文档导航
├── 功能指南.md                  # 👥 用户功能说明（包含所有功能）
├── CHANGELOG.md                 # 📝 版本更新日志
├── dev-guide/                   # 🛠️ 开发文档（12个）
│   ├── README.md
│   ├── 开发指南.md
│   ├── 编译指南.md
│   ├── Makefile使用指南.md
│   ├── 测试指南.md
│   └── ...
└── archive/                     # 📦 历史归档（40+个）
    ├── README.md
    ├── 调试记录
    ├── 实现总结
    └── 测试记录
```

## 整理原则

### 1. 用户文档（根目录）
**保留文档**: 仅 3 个核心文档
- `功能指南.md` - 所有功能的完整说明
- `CHANGELOG.md` - 版本更新日志
- `README.md` - 文档导航

**特点**:
- 面向最终用户
- 保持简洁，易于查找
- 所有功能集中在一个文档中

### 2. 开发文档（dev-guide/）
**包含**: 12 个开发相关文档
- 入门指南（开发、编译）
- 工具使用（Makefile）
- 测试文档
- 发布配置

**特点**:
- 面向开发者和贡献者
- 技术细节和开发流程
- 独立的 README 导航

### 3. 历史归档（archive/）
**包含**: 40+ 个历史文档
- 调试记录
- 实现总结
- 测试记录
- 版本记录
- 临时文档

**特点**:
- 历史参考
- 不再活跃维护
- 独立的 README 说明

## 关键改进

### 合并文档
1. **专注模式使用指南** → 合并到 `功能指南.md`
   - 作为功能指南的第 4 节
   - 包含完整的使用说明和常见问题
   - 避免文档分散

2. **文档整理说明** → 移到 `archive/`
   - 这是一次性的整理记录
   - 归档保存，供参考

### 文档数量对比

| 位置 | 整理前 | 整理后 | 说明 |
|------|--------|--------|------|
| 根目录 | 50+ | 3 | 仅保留核心文档 |
| dev-guide/ | 0 | 12 | 开发文档集中 |
| archive/ | 0 | 40+ | 历史文档归档 |

## 优势

1. **结构清晰**
   - 用户文档：3 个核心文档
   - 开发文档：独立目录
   - 历史文档：归档保存

2. **易于查找**
   - 用户只需看功能指南
   - 开发者查看 dev-guide
   - 历史参考查看 archive

3. **便于维护**
   - 明确的分类标准
   - 清晰的归档原则
   - 完善的导航系统

4. **减少混乱**
   - 从 50+ 减少到 3 个核心文档
   - 其他文档分类存放
   - 不再混在一起

## 维护建议

### 添加新文档
```bash
# 用户功能 → 添加到功能指南.md
# 开发相关 → dev-guide/
# 历史记录 → archive/
```

### 定期整理
- 每个版本发布后归档临时文档
- 功能重构后归档旧文档
- 保持根目录简洁（仅 3 个文档）

### 文档更新
- 新功能添加到功能指南
- 更新 CHANGELOG
- 更新 README 导航

## 总结

通过这次整理：
- ✅ 将 50+ 个文档整理为 3 级分类
- ✅ 根目录仅保留 3 个核心文档
- ✅ 建立完善的导航系统
- ✅ 提升文档可维护性

**核心原则**: 用户文档简洁集中，开发文档分类清晰，历史文档归档保存。
