# 久坐提醒调试指南

## 问题描述
用户报告只能看到喝水提醒，看不到久坐提醒。

## 调试步骤

### 1. 检查当前配置和状态
运行命令：`动了么？: 调试计时器状态`

这会显示：
- 配置状态（久坐提醒是否启用）
- 计时器运行状态
- 提醒函数是否正确注册
- 剩余时间

### 2. 测试短间隔提醒
运行命令：`动了么？: 测试短间隔提醒`

这会：
- 设置1分钟后触发久坐提醒
- 设置1.5分钟后触发喝水提醒
- 验证两个提醒都能正常工作

### 3. 检查控制台日志
打开开发者控制台（Help > Toggle Developer Tools），查看是否有错误信息或调试日志。

### 4. 验证配置
检查VS Code设置中的配置：
- `movedYet.enableSitReminder`: 应该为 `true`
- `movedYet.sitReminderInterval`: 默认60分钟
- `movedYet.enableProgressiveReminder`: 如果启用，检查渐进式提醒是否工作

## 可能的问题和解决方案

### 问题1：久坐提醒被禁用
**解决方案**：在VS Code设置中启用 `movedYet.enableSitReminder`

### 问题2：间隔时间太长
**解决方案**：临时设置较短的间隔进行测试，如5分钟

### 问题3：渐进式提醒问题
**解决方案**：
1. 禁用渐进式提醒：`movedYet.enableProgressiveReminder` 设为 `false`
2. 或检查渐进式提醒服务是否正常工作

### 问题4：活动检测干扰
**解决方案**：
1. 临时禁用活动检测：`movedYet.enableActivityDetection` 设为 `false`
2. 或调整无活动重置时间

### 问题5：提醒函数未正确注册
**解决方案**：
1. 重启VS Code
2. 运行 `动了么？: 强制重启插件`

## 测试结果分析

### 如果测试短间隔提醒成功
说明久坐提醒功能正常，问题可能是：
- 间隔时间太长，还没到触发时间
- 活动检测重置了计时器

### 如果测试短间隔提醒失败
说明久坐提醒功能有问题，需要：
1. 检查控制台错误
2. 重启插件或VS Code
3. 检查代码实现

## 紧急修复命令

如果遇到问题，可以使用以下命令：
- `动了么？: 清理所有提醒（紧急修复）`
- `动了么？: 强制重启插件`
- `动了么？: 重置所有计时器`
