# 智能工作时长测试指南

## 🧪 测试准备

### 1. 安装测试版本
```bash
code --install-extension moved-yet-0.0.2.vsix
```

### 2. 确保配置正确
```json
{
    "movedYet.enableActivityDetection": true,
    "movedYet.inactivityResetTime": 5
}
```

## 🔍 功能测试

### 测试1: 基础状态查看
1. 运行命令：`动了么？: 显示提醒状态`
2. **预期结果**：
   - 显示"工作计时: 进行中"
   - 显示当前会话时长
   - 显示今日工作时长

### 测试2: 活动检测状态
1. 运行命令：`动了么？: 测试活动检测`
2. **预期结果**：
   - 显示活动检测启用状态
   - 显示无活动时长（应该很短）
   - 显示工作计时状态为"进行中"

### 测试3: 10分钟无活动暂停
1. 停止所有编辑活动（不要关闭VS Code）
2. 等待10分钟（可以做其他事情）
3. 回来后运行：`动了么？: 显示提醒状态`
4. **预期结果**：
   - 显示"工作计时: 已暂停"
   - 显示暂停时长（约10分钟）

### 测试4: 活动恢复计时
1. 在编辑器中输入一些文字
2. 立即运行：`动了么？: 显示提醒状态`
3. **预期结果**：
   - 显示"工作计时: 进行中"
   - 显示本次会话时长（应该很短）
   - 今日总工作时长不包含暂停的10分钟

### 测试5: 手动控制
1. 运行：`动了么？: 暂停工作计时`
2. 查看状态，应显示"已暂停"
3. 运行：`动了么？: 恢复工作计时`
4. 查看状态，应显示"进行中"

### 测试6: 仪表盘验证
1. 运行：`动了么？: 查看运动仪表盘`
2. **预期结果**：
   - 今日工作时长只计算实际活跃时间
   - 不包含暂停期间的时间

## 📊 验证要点

### ✅ 正确行为
- 有活动时正常计时
- 10分钟无活动自动暂停
- 恢复活动时自动继续计时
- 暂停期间不累计工作时长
- 手动控制功能正常

### ❌ 错误行为
- 暂停期间仍在累计时长
- 无法检测到活动恢复
- 手动控制无效
- 状态显示错误

## 🐛 常见问题

### Q: 工作计时没有暂停
**A**: 检查活动检测是否启用：
```json
"movedYet.enableActivityDetection": true
```

### Q: 10分钟后没有自动暂停
**A**: 确保完全停止活动：
- 不要移动鼠标
- 不要按键盘
- 不要切换文件
- 不要执行命令

### Q: 恢复后时长计算错误
**A**: 查看控制台日志：
- 按 F1 → "Developer: Toggle Developer Tools"
- 查看 Console 中的日志信息
