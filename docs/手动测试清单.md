# Moved Yet 手动测试清单

## 📋 测试前准备

### 环境准备
- [ ] VS Code 版本 >= 1.100.0
- [ ] Node.js 版本 >= 25.x
- [ ] 关闭其他可能干扰的扩展
- [ ] 备份当前 VS Code 配置

### 安装扩展
```bash
# 安装扩展
code --install-extension moved-yet-0.0.2.vsix

# 验证安装
code --list-extensions | grep moved-yet
```

## 🎯 核心功能测试

### 1. 基础提醒功能

#### 1.1 久坐提醒测试
**配置设置**:
```json
{
    "movedYet.sitReminderInterval": 1,
    "movedYet.enableSitReminder": true,
    "movedYet.enableProgressiveReminder": false
}
```

**测试步骤**:
1. [ ] 保存配置后等待 1 分钟
2. [ ] 观察是否出现全屏久坐提醒
3. [ ] 检查提醒界面文本是否正确
4. [ ] 点击"确认"按钮
5. [ ] 验证提醒消失且计时器重置

**预期结果**: 
- 1分钟后出现全屏提醒
- 界面显示久坐提醒内容
- 确认后提醒消失，重新计时

#### 1.2 喝水提醒测试
**配置设置**:
```json
{
    "movedYet.drinkReminderInterval": 1,
    "movedYet.enableDrinkReminder": true
}
```

**测试步骤**:
1. [ ] 等待 1 分钟
2. [ ] 观察喝水提醒界面
3. [ ] 验证提醒内容正确
4. [ ] 确认提醒功能

**预期结果**: 喝水提醒正常工作

### 2. 渐进式提醒测试

#### 2.1 三级提醒流程
**配置设置**:
```json
{
    "movedYet.enableProgressiveReminder": true,
    "movedYet.progressiveReminderLevel1Duration": 1,
    "movedYet.progressiveReminderLevel2Duration": 1,
    "movedYet.sitReminderInterval": 1
}
```

**测试步骤**:
1. [ ] **第一级 (1分钟后)**: 检查状态栏是否显示提醒图标
2. [ ] **第二级 (2分钟后)**: 检查是否弹出通知消息
3. [ ] **第三级 (3分钟后)**: 检查是否显示全屏提醒
4. [ ] 在第二级时点击"稍后提醒"，验证是否重新开始流程

**预期结果**: 
- 按时间顺序显示三级提醒
- 每级提醒界面正确
- "稍后提醒"功能正常

### 3. 活动检测功能测试 (v0.0.2 新功能)

#### 3.1 基础活动检测
**配置设置**:
```json
{
    "movedYet.enableActivityDetection": true,
    "movedYet.inactivityResetTime": 1
}
```

**测试步骤**:
1. [ ] 运行命令: `Ctrl+Shift+P` -> `动了么？: 测试活动检测`
2. [ ] 检查显示的活动检测状态
3. [ ] 验证无活动时长显示

**预期结果**: 
- 显示"活动检测: 启用"
- 显示当前无活动时长

#### 3.2 自动重置测试
**测试步骤**:
1. [ ] 停止所有操作 1 分钟以上
2. [ ] 在编辑器中输入任意文字
3. [ ] 观察是否出现重置提示消息
4. [ ] 运行状态查看命令，验证计时器是否重置

**预期结果**: 
- 出现"检测到您刚回到工作状态"提示
- 计时器重置为 0

#### 3.3 各种活动类型测试
**测试每种活动类型**:

1. [ ] **键盘输入**: 在编辑器输入文字
   - 运行测试命令，检查无活动时长是否重置

2. [ ] **鼠标点击**: 点击编辑器不同位置
   - 运行测试命令，检查活动是否被检测

3. [ ] **文件切换**: 切换不同文件标签
   - 验证活动检测是否响应

4. [ ] **命令执行**: 运行任意 VS Code 命令
   - 检查活动是否被记录

5. [ ] **文本选择**: 选择编辑器中的文本
   - 验证选择操作是否被检测

**预期结果**: 所有活动类型都能被正确检测并重置无活动时长
